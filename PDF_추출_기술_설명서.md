> 최종 갱신: 2026-04-20

## 1. 개요

본 시스템은 국토지반정보 포털에서 수집한 **시추주상도 PDF 원본**으로부터 지층 데이터를 자동 추출하여 **구조화된 CSV 파일**로 변환하는 파이프라인입니다.

현재 데이터 특성과 정확도 요구에 따라 **3가지 독립적인 변환 경로**가 운용되고 있으며, 세 경로 모두 동일한 후처리 모듈을 공유하여 최종 CSV의 컬럼 규격을 통일합니다.

### 방식 요약

| 구분 | 방식명 | 입력 → 추출 경로 | 적용 대상 |
|:---:|:---|:---|:---|
| **A** | ODL 마크다운 하이브리드 | PDF → Markdown → 정규표현식+공간 폴백 | 서울특별시 |
| **B** | PyMuPDF 인덱스 직접 추출 | PDF → find_tables() → Row/Col 인덱스 | 수원시 (경기도) |
| **C** | HWPX 위치 기반 정밀 추출 | DOCX → HWPX → DataFrame iloc 인덱스 | 신규 문서 대상 |

### 최종 CSV 출력 규격 (공통)

| 컬럼 | 설명 | 예시 |
|:---|:---|:---|
| 프로젝트명 | 파일 기반 고유 식별자 | PJ_105-13 |
| 경도 | X(N) 좌표 | 198234.56 |
| 위도 | Y(E) 좌표 | 452187.89 |
| 표고 | 지표면 표고 (EL, m) | 25.3 |
| 시추공명 | 시추공 기호 | BH-1 |
| 상심도 | 지층 시작 깊이 (m) | 0.0 |
| 하심도 | 지층 종료 깊이 (m) | 2.0 |
| 지층명 | 정규화된 지층 분류 | 매립토 |

- **인코딩**: UTF-8-SIG (엑셀 한글 호환 BOM 포함)

## 2. 방식 A: ODL 마크다운 하이브리드

### 2.1 적용 대상
- 서울특별시 전역 시추주상도 PDF 5,367건
- 산출물: `서울특별시_CSV_통합_최종_V2.csv` (57,019행, 무결성 100%)

### 2.2 처리 흐름

```
[PDF 원본]
    │
    ▼ opendataloader_pdf.convert() — Java 21 기반 엔진
[Markdown 파일] (.md)
    │
    ▼ pdf_parser_odl.extract_all_from_md() — 2-Pass 파싱
    │
    │  ┌─ 1-Pass: PyMuPDF로 각 페이지의 시추공 ID + 좌표/표고 맵 구축
    │  │         (유클리드 거리 기반 공간 근접성 추출)
    │  │
    │  └─ 2-Pass: Markdown 테이블 행을 정규표현식으로 순회
    │            (심도, 지층명 추출 + 심도 리셋 시 시추공 자동 전환)
    │
    ▼ N/A 값이 남은 경우 → PDF 원본 3차 폴백 (ROI 클리핑)
    │
    ▼ table_merger.merge_multi_page_tables()
[최종 CSV]
```

### 2.3 관여 파일

| 파일명 | 크기 | 역할 |
|:---|---:|:---|
| `pdf_parser_odl.py` | 24.8 KB | 핵심 파싱 엔진 (507줄) |
| `batch_processor_v2.py` | 7.2 KB | 5,367건 병렬 배치 처리기 |
| `server.py` | 10.9 KB | Flask 웹 서비스 (실시간 업로드 분석) |
| `table_merger.py` | 7.7 KB | 후처리 (중복제거, 심도보정, 지층병합) |

### 2.4 핵심 기술 상세

#### 유클리드 거리 기반 공간 근접성 추출

PDF 내부의 텍스트는 고정된 테이블 셀이 아니라 **자유 배치된 텍스트 스팬**입니다. 키워드("표고", "X(N)" 등)의 물리적 좌표(bbox)를 앵커로 삼고, 주변의 모든 수치 텍스트까지의 **유클리드 거리를 계산**하여 가장 가까운 수치를 해당 키워드의 값으로 채택합니다.

```python
# pdf_parser_odl.py 핵심 로직 요약
dist = ((anchor_cx - span_cx)**2 + (anchor_cy - span_cy)**2)**0.5
candidates.append((dist, value))
candidates.sort()  # 거리 최소인 값 = 정답
```

#### 손상된 유니코드 앵커 대응

PDF 폰트 인코딩 오류로 한글 키워드가 깨지는 경우에 대응합니다:

| 깨진 문자 | 원래 의미 | 검색 키워드 목록 |
|:---:|:---|:---|
| `ǥ` | 표고 | `["표고", "EL", "ǥ", "\u00A7"]` |
| `浵` | 경도 | `["경도", "X(N)", "浵"]` |
| `ð` | 위도 | `["위도", "Y(E)", "ð"]` |

#### 수치 가드레일

추출된 수치가 의도한 필드에 올바르게 매핑되었는지 범위 검증합니다:

| 필드 | 유효 범위 | 목적 |
|:---|:---|:---|
| 좌표 (경도/위도) | 100,000 ~ 1,000,000 | 표고(25.3)가 좌표로 오매핑되는 것 방지 |
| 표고 | -100 ~ 5,000 | 좌표(198234)가 표고로 오매핑되는 것 방지 |

## 3. 방식 B: PyMuPDF 인덱스 직접 추출

### 3.1 적용 대상
- 경기도 수원시 3개 구 (권선구, 장안구, 팔달구)
- 산출물: 구별 `_Stage46.csv` 파일 3건

### 3.2 처리 흐름

```
[PDF 원본]
    │
    ▼ fitz.open() → page.find_tables() — PyMuPDF 표 자동 인식
    │
    ├─ tables[0].extract() → 메타데이터 표 행렬
    │   ├─ [0][7] = 시추공명
    │   ├─ [1][5] = 좌표 X(N)/Y(E)
    │   └─ [1][7] = 표고
    │
    └─ tables[1].extract() → 지층 데이터 표 행렬
        └─ Row 2~ 순회: [row][0]=하심도, [row][4]=지층명
    │
    ▼ 상심도/하심도 계산 및 포맷팅
[최종 CSV]
```

### 3.3 관여 파일

| 파일명 | 크기 | 역할 |
|:---|---:|:---|
| `hwp_indexed_extractor.py` | 8.6 KB | 인덱스 기반 추출 엔진 |
| `hwp_batch_runner.py` | 5.0 KB | 수원시 3개 구 병렬 배치 |

### 3.4 표 구조 매핑 규칙

시추주상도 PDF는 페이지당 2개 표로 구성됩니다:

```
┌──────────────────────────────────────────────┐
│  Table 0 (메타데이터)                          │
│  Row 0: [...] [0,7]=시추번호                   │
│  Row 1: [...] [1,5]=좌표  [...] [1,7]=표고     │
├──────────────────────────────────────────────┤
│  Table 1 (지층 데이터)                          │
│  Row 0-1: 헤더                                │
│  Row 2~:  [row,0]=심도  [row,4]=지층명         │
└──────────────────────────────────────────────┘
```

### 3.5 방식 A 대비 차이

| 비교 항목 | 방식 A | 방식 B |
|:---|:---|:---|
| 중간 변환 | PDF → Markdown 필요 | 없음 (직접 추출) |
| 외부 의존성 | Java 21 (ODL) | 없음 |
| 추출 방식 | 정규표현식 + 공간 좌표 | 인덱스 직접 접근 |
| 처리 속도 | 중간 | **빠름** |


## 5. 공통 후처리 모듈: table_merger.py

세 가지 방식 모두 추출 이후 동일한 3단계 후처리를 거칩니다.

### 5.1 처리 순서

| 단계 | 함수명 | 역할 |
|:---:|:---|:---|
| 1 | `deduplicate_overlap_rows()` | (시추공명, 상심도, 하심도) 키 기반 중복 행 제거 |
| 2 | `apply_depth_continuity_correction()` | 이전 행의 하심도를 다음 행의 상심도와 일치시켜 연속성 보장 |
| 3 | `merge_consecutive_identical_soil_types()` | 같은 시추공의 연속 동일 지층을 하나로 병합, 두께 재계산 |

### 5.2 보정 예시

```
[보정 전]
BH-1  토사   0.0 ~ 2.0
BH-1  풍화토  1.5 ~ 5.5   ← 상심도(1.5)가 이전 하심도(2.0)보다 작음 = 중첩

[보정 후]
BH-1  토사   0.0 ~ 2.0
BH-1  풍화토  2.0 ~ 5.5   ← 상심도를 이전 하심도(2.0)로 강제 보정
```
