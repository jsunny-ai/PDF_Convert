# 🏗️ GeoBIM 시추 데이터 변환 파이프라인 기술 설명서 (v1.0)

본 문서는 수원시 시추공 주상도 PDF 파일을 고순도 CSV 데이터로 변환하기 위해 설계된 **4단계 하이브리드 추출 엔진(Stage 52 규격)**의 아키텍처와 상세 작동 원리를 기술합니다.

---

## 1. 시스템 아키텍처 개요

본 파이프라인은 데이터 유실을 최소화하고 정합성을 극대화하기 위해 다중 계층(Multi-Tier) 분석 구조를 채택하고 있습니다.

### 🔄 4단계 하이브리드 워크플로우
1.  **Tier 1: HWPX 원본 추출 (Direct Access)**
    *   PDF 내에 포함된 HWPX 표 오브젝트를 직접 탐색하여 데이터 손실 없이 표 구조를 복구합니다.
2.  **Tier 1.5: 인덱스 기반 정밀 추출 (Coordinate Target)**
    *   PyMuPDF를 사용하여 PDF 페이지 내의 셀(Cell) 좌표를 직접 계산합니다.
    *   텍스트의 공간적 배치를 분석하여 행(Row)과 열(Column)을 재구성하며, OCR 오인식 없는 순수 텍스트 추출을 수행합니다.
3.  **Tier 3: 논리 구조 분석 (ODL Integration)**
    *   OpenDataLoader를 활용하여 페이지 경계에서 잘린 표를 하나로 연결합니다.
    *   시추 정보(좌표, 표고)와 지반 정보(심도, 지층명) 간의 논리적 연결 고리를 생성합니다.
4.  **Stage 4: 지능형 데이터 병합 (Smart Merger)**
    *   중복 데이터를 제거하고, 파편화된 지층 정보를 심도 순서에 따라 연속적으로 결합합니다.

## 2. 핵심 데이터 처리 로직

### 📍 2.1 좌표계 정합성 및 공간 분석
*   **WGS84 단일화**: 수원시 지역 TM 좌표계를 표준 경위도(EPSG:4326)로 변환합니다.
*   **방어적 스왑 체크**: 경도(Longitude)와 위도(Latitude)가 뒤바뀌어 추출되는 현상을 방지하기 위해 수원시 경계 박스(Bounding Box)를 기준으로 수치 검증을 수행합니다.
*   **결측치 보존**: 좌표 정보가 없는 행이라도 지반 정보가 유효할 경우 데이터를 삭제하지 않고 유지합니다.

### 🧪 2.2 지층 분류 및 심도 병합
*   **4대 대분류 정규화**: 수만 가지의 가변적인 지층명을 시스템 표준인 **[토사, 풍화암, 연암, 경암]**으로 자동 분류합니다.
    *   *토사*: 매립토, 퇴적토, 풍화토 포함
    *   *연암*: 연암, 리핑암 포함
    *   *경암*: 보통암, 경암, 발파암, 화강암 포함
*   **심도 연속성 보정**: 상위 지층의 하심도와 하위 지층의 상심도가 일치하지 않을 경우(Overlapping), 논리적 오차를 0.01m 단위로 정밀 보정합니다.

### 🆔 2.3 전역 정합성 (Single Source of Truth)
*   **프로젝트명 브로드캐스팅**: 하위 파서에서 생성되는 임시 식별자(PJ_ 등)를 배제하고, 마스터 엔진이 부여한 원본 파일명 기반 프로젝트명을 모든 데이터 행에 강제 동기화합니다.
*   **자동 넘버링**: 시추번호(BH ID)가 누락된 경우, 동일 프로젝트 내 순번에 따라 유니크한 ID를 자동 생성하여 데이터 유실을 방지합니다.

## 3. 성능 및 운영 최적화

### ⚡ 3.1 분석 결과 캐싱 (Caching Strategy)
*   ODL 분석은 CPU 집약적인 작업이므로, 분석된 마크다운 결과를 `data/02_markdown` 폴더에 영구 저장합니다.
*   동일 파일에 대한 재요청 시 연산 과정을 생략하고 캐시 데이터를 로드하여 배치 작업 속도를 **최대 20배 이상** 향상시켰습니다.

### 📊 3.2 배치 처리 아키텍처
*   **역할 분담**: 수집 프로젝트와 변환 프로젝트를 엄격히 분리하여, 변환 엔진은 로컬 파일 시스템에 준비된 PDF의 분석에만 100% 자원을 집중합니다.
*   **예외 처리**: 파일 잠금(Permission Error) 발생 시 자동으로 버전 관리(`v2.csv`)를 수행하여 작업 중단 없는 대규모 배치를 지원합니다.

## 4. 최종 산출물 규격

생성되는 모든 통합 CSV 파일은 다음의 표준 컬럼 스키마를 준수합니다:

| 컬럼명 | 설명 | 비고 |
| :--- | :--- | :--- |
| **프로젝트명** | 원본 파일명 기반 식별자 | SSOT 적용 |
| **lon_wgs84** | 경도 (WGS84) | 수치 검증 완료 |
| **lat_wgs84** | 위도 (WGS84) | 수치 검증 완료 |
| **표고** | 시추 지점의 해발 고도 | - |
| **시추공명** | Borehole ID | 정규화 완료 |
| **상심도** | 지층의 시작 깊이 | 단위: m |
| **하심도** | 지층의 종료 깊이 | 단위: m |
| **지층명** | 4대 표준 지층 분류 | 정규화 완료 |
